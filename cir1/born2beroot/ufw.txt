UFW 주요 설정 파일 위치
기본 설정 파일
/etc/default/ufw

기본 정책, IPv6 지원, 사용할 커널 모듈 등 고수준 설정이 포함됩니다

여기서 DEFAULT_FORWARD_POLICY를 "ACCEPT"로 설정하여 포워딩 정책을 변경할 수 있습니다

/etc/ufw/ufw.conf

부팅 시 UFW 활성화 여부 설정 및 로그 레벨(LOGLEVEL) 설정이 포함됩니다

ENABLED=yes/no로 UFW 활성화 여부를 설정할 수 있습니다

규칙 설정 파일
/etc/ufw/before.rules

UFW 명령어로 추가된 규칙보다 먼저 평가되는 규칙이 포함됩니다

NAT, IP Masquerade 설정을 이 파일에 추가할 수 있습니다

/etc/ufw/after.rules

UFW 명령어로 추가된 규칙 이후에 평가되는 규칙이 포함됩니다

/var/lib/ufw/user.rules 또는 /lib/ufw/user.rules (0.28 이상 버전)

UFW 명령어를 통해 추가된 규칙이 저장됩니다(일반적으로 직접 편집하지 않음)

시스템 설정 파일
/etc/ufw/sysctl.conf

커널 네트워크 튜닝 설정이 포함됩니다

IP 포워딩 활성화와 같은 설정을 이 파일에서 할 수 있습니다:

text
net/ipv4/ip_forward=1
이 설정은 /etc/sysctl.conf의 설정을 오버라이드합니다

UFW 설정 예시
기본 명령어로 설정 변경
대부분의 UFW 설정은 명령줄 인터페이스를 통해 수행됩니다:

bash
sudo ufw enable        # 방화벽 활성화
sudo ufw disable       # 방화벽 비활성화
sudo ufw status        # 방화벽 상태 확인
sudo ufw allow 22/tcp  # SSH 포트 허용
이러한 명령어로 설정한 규칙은 /var/lib/ufw/user.rules 파일에 자동으로 저장됩니다.

NAT 및 포워딩 설정
라우터나 방화벽으로 Ubuntu를 구성할 경우 /etc/ufw/before.rules 파일을 수정하여 NAT 설정을 추가할 수 있습니다:

text
*nat
:POSTROUTING ACCEPT [0:0]
-A POSTROUTING -s 192.168.10.0/24 -o enp3s0 -j MASQUERADE
COMMIT
설정 변경 후 적용
UFW 설정 파일을 직접 수정한 후에는 다음 명령어로 설정을 다시 로드해야 합니다:

bash
sudo ufw reload
또는 서비스를 재시작할 수도 있습니다:

bash
sudo systemctl restart ufw
UFW는 iptables를 기반으로 하지만 더 간단한 인터페이스를 제공하여 방화벽 관리를 용이하게 합니다. 복잡한 방화벽 구성이 필요한 경우 iptables 규칙을 직접 사용해야 할 수도 있습니다
