SELinux(Security-Enhanced Linux)는 리눅스 커널에 포함된 MAC(Mandatory Access Control) 시스템으로, 기존의 DAC(Discretionary Access Control) 방식보다 더 강력한 보안 정책을 적용할 수 있도록 설계된 보안 모듈입니다.

✅ 1. SELinux의 목적
기본적인 리눅스 보안(DAC)은 파일 소유자나 프로세스가 권한을 마음대로 바꿀 수 있습니다. 하지만 SELinux는 이런 행동조차 **정책(policy)**에 따라 엄격하게 제한합니다.

예를 들어:

일반적으로 root는 모든 걸 할 수 있지만,

SELinux에서는 정책이 허용하지 않으면 root라도 접근이 거부됩니다.

✅ 2. 주요 개념
개념	설명
MAC (Mandatory Access Control)	시스템 차원에서 접근 제어 정책을 강제함. 유저가 임의로 권한 변경 불가
정책(Policy)	어떤 주체(subject)가 어떤 객체(object)에 어떤 작업을 할 수 있는지 명시
주체(subject)	프로세스나 사용자 등
객체(object)	파일, 디렉토리, 포트, 소켓 등 자원
컨텍스트(Context)	각 주체와 객체가 가지는 SELinux 속성
user:role:type:level 형태
도메인(Domain)	주체에 적용되는 정책의 묶음 (보통 type)

✅ 3. 작동 방식 요약
시스템에 있는 모든 파일/프로세스에는 **보안 컨텍스트(context)**가 부여됨

✅ 4. 동작 모드
모드	설명
Enforcing	정책을 실제로 적용 (차단 발생)
Permissive	정책 위반은 허용하되, 로그 기록만 함
Disabled	SELinux 완전 비활성화

항목	내용
SELinux는 UID가 root라도 정책이 막으면 접근 거부	
파일과 실행파일은 서로 다른 SELinux type을 가짐	
정책이 허용하지 않으면 접근 불가 (even with chmod 777)	
chcon으로 컨텍스트 임시 변경 가능	
restorecon으로 기본값으로 되돌림	
거부 로그는 /var/log/audit/audit.log에 저장됨
