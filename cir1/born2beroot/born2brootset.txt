			PASS_MAX_DAYS   99999

		참고
		/etc/login.defs 파일은 리눅스 시스템에서 사용자 계정 생성 및 로그인 정책을 정의하는 전역 설정 파일입니다.
			이 파일은 useradd, passwd, login 같은 명령어들이 사용할 기본값과 보안 정책들을 설정하는 데 사용됩니다.
			/etc/login.defs에 담기는 주요 정보
		항목	설명
		PASS_MAX_DAYS	비밀번호 최대 사용일수 (예: 30 → 30일 후 만료)
		PASS_MIN_DAYS	비밀번호 변경 최소 대기일 (예: 2 → 2일 이내 재변경 금지)
		PASS_WARN_AGE	비밀번호 만료 경고일 (예: 7 → 만료 7일 전부터 경고)
		UID_MIN, UID_MAX	일반 사용자 계정의 UID 범위
		GID_MIN, GID_MAX	일반 그룹의 GID 범위
		CREATE_HOME	useradd 시 홈 디렉토리 자동 생성 여부 (yes or no)
		ENCRYPT_METHOD	비밀번호 해싱 알고리즘 (SHA512, MD5, 등)
		LOGIN_RETRIES	로그인 시도 실패 허용 횟수 (지원 여부는 PAM에 따라 다름)
		LOGIN_TIMEOUT	로그인 시 타임아웃 (초 단위)
		UMASK	기본 파일 권한 마스크 설정 (ex. 022)
		USERGROUPS_ENAB	사용자 삭제 시 관련 그룹도 삭제할지 여부
		
		1-0, 1-1중 하나 선택
		1-0)/etc/security/pwquality.conf 에 설정 추가 // 이건 비추
			dcredit = -1     # 숫자 1개 필수
			ucredit = -1     # 대문자 1개 필수
			lcredit = -1     # 소문자 1개 필수
			....
			기본 정책 적용후 /etc/pam.d/common-password을 1-1과 비슷하게 알맞게 수정
			1-1)/etc/pam.d/common-password을 수정
			- 바꿀려는 유저가 root일경우 한줄 스킵(success=1)// success=1 뿐만아니라 done 등 다양함 // user, uid 적용방법도 다양함
			password        [success=1 default=ignore]      pam_succeed_if.so   user = root
			password        requisite       pam_pwquality.so        enforce_for_root \
												retry=3 \
												dcredit=-1 ucredit=-1 ocredit=0 \
												minlen=0
			- 바꿀려는 유저가 root가 아닐 경우 한줄 스킵(success=1)
			password        [success=1 default=ignore]      pam_succeed_if.so   user != root
			password        requisite       pam_pwquality.so        enforce_for_root \
												retry=3 \
												dcredit=0 ucredit=0 ocredit=-1 \
												minlen=0
			>> 루트 유저와 일반 유저에게 다른 정책 적용 // 
			참고. difok의 경우 루트로 로그인 할 경우 무시됨, 일반유저는 적용됨
				passwd로 비밀번호 변경시 루트는 현재 비밀번호 요청 없이 바로 진행
				하지만 일반유저의 경우 현재 비밀번호 요청 후 변경진행
				> 즉, 루트는 이전 비밀번호를 모르기 때문에 difok적용이 불가
