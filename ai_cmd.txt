[FINAL_PROJECT_SUMMARY]

프로젝트명
- UART + CAN 기반 사용자/관리자 제어 시스템

프로젝트 성격
- 2주 정도의 토이 프로젝트
- 구조적 안정성 / 적절한 확장성 / 구현 가능성 중심
- RTOS 없이 cooperative task scheduler 기반
- UART / CAN / Device / State 분리 구조

10. 핵심 설계 원칙
==================================================================

분리 원칙
- UART = 문자열 입출력
- CAN = 프레임 송수신
- Device = 장치 상태 / 인증 / 이벤트
- Node = 전체 흐름 제어
- Task = 시간 기반 실행 관리

구조 원칙
- Node A / Node B 역할 분리
- UART / CAN / Device 책임 분리
- 상태머신 기반 제어
- 태스크 기반 주기 실행
- 동적 할당 없이 정적 구조 중심
- 2주 프로젝트에 맞는 경량 구조 유지


==================================================================
11. AI 입력용 핵심 요약
==================================================================

- This project has two nodes: Node A and Node B.
- Node A is a user request gateway.
- Node B is an admin control + device management node.
- System flow is: Putty A <-> UART <-> Node A <-> CAN <-> Node B <-> UART <-> Putty B.
- Node A handles user UART input, request generation, CAN transmit, CAN response receive, and user output.
- Node B handles CAN request receive, request parsing, device/password validation, admin UART input, admin command processing, local device control or forwarding to Node A, and admin output/logging.
- Scheduler is cooperative and task-based with 10ms / 50ms / 100ms / 500ms periodic tasks.
- Common structs are UART채널정보, CAN메시지, CAN채널정보, 접근요청정보, 접근응답정보, 장치정보, 장치테이블, 태스크정보.
- Top-level node structs are 노드A정보 and 노드B정보.
- Architecture separates UART, CAN, Device, Node state, and Task scheduling.
