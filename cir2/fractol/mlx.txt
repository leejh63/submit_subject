mlx 라이브러리 콜백 함수 인자 형태 총정리
아래 표는 MiniLibX(mlx) 라이브러리에서 이벤트를 처리할 때 사용하는 주요 hook 함수와, 각 콜백 함수가 받는 인자 형태를 한눈에 정리한 것입니다. 모든 주요 이벤트와 함수의 인자 구조를 빠짐없이 포함했습니다.

Hook 함수명	콜백 함수 형태	주요 인자 설명
mlx_key_hook	int key_hook(int keycode, void *param)	keycode: 눌린 키의 코드
param: 사용자 포인터
mlx_mouse_hook	int mouse_hook(int button, int x, int y, void *param)	button: 마우스 버튼 번호
x, y: 클릭 좌표
param: 사용자 포인터
mlx_expose_hook	int expose_hook(void *param)	param: 사용자 포인터
mlx_loop_hook	int loop_hook(void *param)	param: 사용자 포인터
mlx_hook	이벤트에 따라 다름 (아래 참고)	x_event, x_mask에 따라 인자 구조가 달라짐
상세 설명
mlx_key_hook

키보드 입력 이벤트를 처리

콜백 함수는 반드시 (int keycode, void *param) 형태여야 함.

mlx_mouse_hook

마우스 클릭 이벤트를 처리

콜백 함수는 반드시 (int button, int x, int y, void *param) 형태여야 함.

mlx_expose_hook

창이 새로 그려질 때(노출 이벤트) 호출

콜백 함수는 반드시 (void *param) 형태여야 함.

mlx_loop_hook

이벤트가 없어도 루프마다 호출

콜백 함수는 반드시 (void *param) 형태여야 함.

mlx_hook

X11의 다양한 이벤트를 처리할 수 있는 범용 함수

x_event에 따라 콜백 함수의 인자 형태가 다름

예시: 키보드 이벤트면 (int keycode, void *param), 마우스 이벤트면 (int button, int x, int y, void *param) 등.

참고 사항
모든 hook 함수는 마지막 인자로 void *param을 받아, 원하는 구조체나 데이터를 콜백에서 사용할 수 있도록 함.

콜백 함수의 인자 개수와 타입은 반드시 해당 hook 함수의 요구사항에 맞춰야 하며, 다를 경우 컴파일러 경고 또는 런타임 오류가 발생할 수 있음.

mlx_hook 함수는 X11 이벤트 번호와 마스크에 따라 다양한 형태의 콜백을 지원하지만, 대표적으로는 위와 같은 인자 구조를 사용함.

요약
콜백 함수 인자는 이벤트별로 완전히 고정되어 있다.

*모든 hook 함수의 마지막 인자는 void param으로, 사용자 데이터 전달에 사용된다.

인자 형태를 임의로 바꾸면 동작하지 않는다.

이 표와 설명을 참고하면, mlx에서 사용할 수 있는 모든 콜백 함수의 인자 구조를 한눈에 파악할 수 있습니

1. void *mlx_init(void);

반환값: 성공 시 핸들러 포인터, 실패 시 NULL.
