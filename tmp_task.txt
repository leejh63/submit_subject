프로젝트명
- UART + CAN 기반 사용자/관리자 제어 시스템

핵심 구성
- Node A : 사용자 요청 처리 노드
- Node B : 관리자 제어 + 장치 관리 노드

전체 흐름
- Putty A <-> UART <-> Node A <-> CAN <-> Node B <-> UART <-> Putty B

Node A 역할
- 사용자 입력 수신
- 요청 생성
- CAN 송신
- 응답 수신
- 사용자 출력

Node B 역할
- CAN 요청 수신
- 장치 검사 / 인증
- 관리자 입력 수신
- 관리자 명령 처리
- 로컬 장치 처리 또는 Node A 전달
- 관리자 출력 / 로그 출력

실행 구조
- 시스템초기화()
- 모듈초기화()
- 서비스초기화()
- 노드초기화()
- 태스크초기화()
- while(1) -> 노드별태스크스케줄러실행()

태스크 구조
{
    공통 주기
    - 10ms
    - 50ms
    - 100ms
    - 500ms

    Node A 태스크
    - 10ms : UART입력처리 / CAN수신처리 / 요청상태처리 / 응답대기처리 / 타임아웃처리
    - 50ms : 상태LED갱신 / 통신상태표시갱신
    - 100ms : UART출력처리 / 사용자메시지갱신 / 오류상태점검
    - 500ms : 생존상태갱신 / 디버그상태점검

    Node B 태스크
    - 10ms : UART입력처리 / CAN수신처리 / 사용자요청처리 / 관리자명령처리 / 장치상태처리 / 타임아웃처리
    - 50ms : 장치LED갱신 / 그룹LED갱신 / 장치상태표시갱신
    - 100ms : UART출력처리 / 관리자메시지갱신 / 이벤트로그갱신 / 오류상태점검
    - 500ms : 생존상태갱신 / 장치요약상태갱신 / 디버그상태점검
}
